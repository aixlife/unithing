## 핵심 방향 (2026-04-24 확정)

**"사후설계"가 아닌 "사전설계" 서비스**

9등급→5등급제로 등급 구간이 넓어지면서, 이제 입시는 결과 분석보다 **목표를 정하고 거기서 역으로 설계하는 것**이 핵심이 됨.

### 서비스가 해야 할 것

**기존 생각 (진단):**
- 생기부 분석 → "너는 이 대학 수준이야"

**실제 가치 (처방+로드맵):**
1. 생기부 분석 → 현재 수준의 대학들 추천
2. **"이 대학에 가려면 지금부터 이걸 해야 해"** (역설계)
   - 어떤 과목을 선택해야 하는지
   - 세특 활동을 어떤 주제로 해야 하는지
   - 생기부에서 아쉬운 부분은 무엇이고 어떻게 보완할지

### 이동준 국장님 표현
> 포커싱을 맞춰야 하는 부분은 생기부를 분석해서 '너는 이 대학이야'라는 것도 당연히 나와야 하지만,
> 지금 너의 생기부를 분석해보니 이런 대학들이 추천이 되는데,
> **그 대학들을 가기 위해서는 과목설계라던지 세특활동들을 어떤 것들을 해야 하는지가 이 프로그램의 핵심**

### 김강석 선생님 표현
> 5등급으로 대학 가기 위해서는 현재는 이 정도인데,
> **더 좋은 대학을 가기 위해서는 이런 과목을 선택해야 하고**,
> 생기부도 좋아야 하는데, 네 생기부를 분석하니 아쉬운 부분이 있네.
> 이를 보완하기 위해서는 → **필요한 교과세특은 이런 주제로 하면 좋을 것 같아**

## How to apply (개발 우선순위에 반영)

- 모든 서비스를 **연결된 흐름**으로 설계해야 함
  - Service3 (생기부 분석) → Service1 (대학 찾기/매칭) → Service2 (과목 가이드) → Service4 (세특 주제 추천)
- 각 서비스가 독립 기능이 아닌 **"생기부 → 목표 대학 → 역설계 경로"** 한 흐름의 단계
- Service1의 핵심은 단순 검색이 아니라 "내 생기부 기반 대학 추천 + 목표 대학 입력 → 필요 스펙 역산"

**Why:** 9등급→5등급제로 변경된 입시 환경에서 사전 설계가 경쟁력. 이게 서비스 차별점이자 선생님들이 학생 상담에서 실제로 필요한 것.
